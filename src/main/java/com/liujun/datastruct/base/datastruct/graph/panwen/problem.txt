描述
计算系统中共有N个任务，任务之间可能存在相互依赖（不存在自己到自己的依赖）。
每个任务由10个以内的大写字母（'A'-'Z')作为标识，依赖关系由'->'描述。
例如
ABC->DDD
表示任务ABC的开始依赖于DDD的完成。
现在需要判断是否存在一种任务安排，可以顺利的做完所有任务。
输入
输入的第一行有2个整数N(0 < N <= 1000)，M(0 < M <= 1000000)，表示该项目中存在N个任务，以及M个依赖关系。
以下有M行，每一行有一个合法的任务依赖关系的表述。
输出
输出为一行，如果当前存在一种安排，可以遵照依赖关系完成所有任务，则输出"YES"，否>则输出"NO"（均为大写字母）
输入样例
3 3
AA->B
AA->C
B->C
输出样例
YES
